# Intervo.ai: Application Setup Guide

This guide will walk you through setting up and running the **Intervo.ai** application using Docker Compose in a local development environment.

---

## 📦 Prerequisites

Before you start, ensure the following software is installed:

### Required Tools

- **Docker & Docker Compose**  
  Install [Docker Desktop](https://www.docker.com/products/docker-desktop) (includes Docker Engine + Compose).

- **Git**  
  [Install Git](https://git-scm.com/downloads) for your operating system.

- **FFmpeg**  
  Required for audio processing.

#### FFmpeg Installation

**macOS (Homebrew)**  
```bash
brew install ffmpeg
```

**Ubuntu/Debian**  
```bash
sudo apt update && sudo apt install ffmpeg
```

**Windows**  
```bash
choco install ffmpeg
# or
scoop install ffmpeg
```

## 🚀 Setup Instructions

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd intervo-react
```

### 2. Verify Directory Structure

Ensure the structure matches the layout above. Docker volumes and npm workspaces depend on this.

### 3. Start the Application

**Foreground (with logs)**

```bash
docker-compose -f opensource-docker-compose.yml up
```

**Detached Mode**

```bash
docker-compose -f opensource-docker-compose.yml up -d
```

### 4. First-Time Setup

On the first run, Docker will:

- Pull base images
- Install npm and pip dependencies
- Initialize MongoDB

**Startup Order:**

1. MongoDB
2. Backend (after MongoDB is ready)
3. Frontend and RAG API (in parallel)

## 🛠 Development Workflow

### Code Changes

- **Frontend**: Auto-reloads via Next.js
- **Backend**: Auto-restarts via `nodemon`
- **RAG API**: Restart container manually after changes

### View Logs

```bash
# All logs
docker-compose -f opensource-docker-compose.yml logs

# Specific service
docker-compose -f opensource-docker-compose.yml logs backend

# Real-time logs
docker-compose -f opensource-docker-compose.yml logs -f
```

## 🧪 Managing the Application

### Stop Services

```bash
docker-compose -f opensource-docker-compose.yml down
```

### Clean Reset (Remove Volumes)

```bash
docker-compose -f opensource-docker-compose.yml down -v
```

### Rebuild Services

```bash
docker-compose -f opensource-docker-compose.yml up --build
```

### Restart Specific Service

```bash
docker-compose -f opensource-docker-compose.yml restart frontend
```

## 🧩 Troubleshooting

### Common Issues

- **Port Conflicts**  
  Check with: `sudo lsof -i :3000`

- **Docker Stuck**  
  Restart Docker or run: `docker system prune`

- **Corrupt Volumes**  
  Run: `docker-compose -f opensource-docker-compose.yml down -v`

- **Build Failures**  
  Clean and rebuild:
  ```bash
  docker-compose -f opensource-docker-compose.yml down -v
  docker-compose -f opensource-docker-compose.yml up --build
  ```

### Debugging Services

```bash
# Container status
docker-compose -f opensource-docker-compose.yml ps

# Open shell in backend
docker-compose -f opensource-docker-compose.yml exec backend sh
```

## 💡 Development Tips

- **Live Reload**: Code changes reflect instantly due to mounted volumes
- **Persistent MongoDB**: Data is stored in Docker volume `mongodb_data`
- **Clean Fix**: If stuck, run:
  ```bash
  docker-compose -f opensource-docker-compose.yml down -v && docker-compose -f opensource-docker-compose.yml up
  ```
